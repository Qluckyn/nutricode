# 手部营养状态合成数据扩增与质量筛选快速实验规划书

## 1. 目标与研究问题

本实验以仅真实 pose02 数据训练的 `C0-pose02` 为基础，快速验证以下两个问题：

1. 在受试者级严格隔离下，加入合成手部图像是否优于仅使用真实图像训练？
2. 在合成图数量、营养类别、姿势比例和训练优化步数一致时，经质量筛选的合成图是否优于从同一候选池随机抽取的原始合成图？

本轮只用于路线筛选，所有结果标记为 `fold_0 + seed22` 的探索性证据；不得据此报告临床诊断性能或正式泛化结论。

## 2. 冻结范围

| 项目 | 冻结设置 |
|---|---|
| 外层折 | `fold_0` |
| 数据划分 | `/root/autodl-tmp/data_hand/pose01_pose02_5fold_seed22/split.json` |
| 分类训练种子 | `22` |
| 真实主 baseline | `C0-pose02`：仅真实 `pose02` 图 |
| 外层测试集 | 仅在最终冻结方案完成训练后评估一次；不参与LoRA、父图、条件、QC、阈值或参数选择 |
| 分类模型 | CLIP ViT-B/16，图像侧 LoRA；文本侧 LoRA 关闭 |
| 分类预处理 | `v2c_deterministic`；不使用 MixUp/CutMix |
| 分类训练 | 40 epochs，batch size 8，learning rate `1e-5`，warmup 4 epochs，weight decay `1e-4`，固定最后一轮 checkpoint |
| 生成路线 | FoundHand 结构锚定 + 当前折 pose02 类别 LoRA + SD2.1 OpenPose ControlNet 低强度 img2img |

## 3. 数据隔离与标签边界

1. LoRA训练图、外观父图、FoundHand结构条件、QC真实参考集和分类训练图均只能来自 `fold_0/train` 受试者。
2. 本实验只使用 `pose02`；同一受试者的 `pose02` 及其所有合成后代必须属于同一数据分区。
3. `fold_0/test` 的图像、关键点、父图、类别比例、预测、错误案例和评价指标均不得用于生成、筛选、调参或选模。
4. 合成图的营养类别是由同类真实外观父图继承的候选标签，不构成新的临床真值。不得仅凭提示词宣称表型或营养状态已被证明保留。

## 4. 生成资产准备

### 4.1 类别 LoRA

训练两个 `fold_0` 专用 LoRA：

```text
malnourished_hand：该折营养不良训练受试者的 pose02 真实图
normal_hand：该折营养正常训练受试者的 pose02 真实图
```

姿势由 FoundHand 的双手关键点条件和 OpenPose ControlNet 控制，不按姿势拆分 LoRA。所有训练输入、权重路径、哈希和受试者清单写入审计报告。

### 4.2 结构条件池

仅为 `pose02` 建立至少12套、来自当前训练折的人工确认双手结构条件。每套条件记录：

```text
condition_id、pose、关键点、掩码、结构供体受试者、来源路径、审核状态、哈希
```

结构条件在两类间对称复用；不得让类别与姿势、背景、构图或条件供体绑定。

### 4.3 生成计划

生成前创建不可变的 `generation_plan.jsonl`，每一行至少包含：

```text
fold、nutrition_class、pose、appearance_parent_subject_id、appearance_parent_path、
condition_id、condition_donor_subject_id、generation_seed、LoRA路径、
FoundHand权重哈希、ControlNet权重哈希、strength、CFG、steps、输出路径
```

## 5. 大规模候选生成

对每个“营养类别 × 姿势”组合生成3,000张候选：

```text
2 类 × 1 姿势（pose02）× 3,000 = 6,000 张候选
```

建议初始冻结生成参数：

```text
base model：SD2.1
结构模型：FoundHand + OpenPose ControlNet
模式：同类真实父图 img2img
denoising strength：0.15
guidance scale：3.5
inference steps：30
resolution：512 × 512
```

每名外观父受试者的候选数设置上限，并使类别、姿势、父受试者与结构条件的贡献尽量均衡。若固定参数在训练折内部的预先定义校准未通过，不得根据外层测试结果调整。

## 6. 质量筛选

### 6.1 基础有效性（S1）

- 文件可读、分辨率正确、非空白、非低方差；
- 无严重模糊、异常曝光或损坏；
- 元数据、生成计划和图像文件一一对应。

### 6.2 结构与姿势（S2）

- 恰好检测到两只手；
- 双手关键点可解析；
- 目标 `pose02` 一致；
- 无明显裁切、额外手、缺失手或严重手指解剖异常。

### 6.3 近重复与分布约束（S3）

- 排除与外观父图近乎复制的候选；
- 仅以 `fold_0/train` 真实图建立冻结的特征参考分布；
- 排除明显偏离同类真实参考分布的候选；
- 审计背景、亮度、色彩、清晰度和前景占比，避免类别捷径；
- 限制单一父受试者对最终池的贡献。

筛选不得使用下游分类器在外层测试集上的预测或性能。所有阈值、特征模型版本和筛选原因逐图写入 `qc_manifest.jsonl`。

## 7. 合成训练池构建

从同一个6,000张候选宇宙中构建两套严格等量训练池：

```text
C1-raw：每个“类别 × 姿势”从通过S1的候选中随机抽取1,500张
C2-qc ：每个“类别 × 姿势”从通过S1、S2、S3的候选中确定性选择1,500张
```

每套合成训练池大小：

```text
2 类 × 2 姿势 × 1,500 = 6,000 张合成图
```

若任一组合无法获得1,500张S1-S3合格候选，停止实验并报告，不降低阈值、不改用外层测试信息补选。

`C1-raw` 与 `C2-qc` 必须匹配：

- 合成图总数；
- 营养类别数；
- `pose02` 数量；
- 父受试者贡献上限；
- 生成参数覆盖；
- 分类采样、batch、epoch、学习率和随机种子。

## 8. 三组分类对照

所有组均训练40 epochs、batch size 8，并固定最后一轮checkpoint。`C1-raw-3x`、`C2-qc-3x` 的真实:合成曝光比例固定为 `1:3`；两组合成图数量、类别、姿势和训练步数严格一致。

| 组别 | 每轮真实图 | 每轮合成图 | 每轮总量 | 每轮优化步数 | 总优化步数 |
|---|---:|---:|---:|---:|---:|
| `C0-pose02` | 24张 | 0张 | 24张 | 3 | 120 |
| `C1-raw-3x` | 24张 | 72张随机原始合成图 | 96张 | 12 | 480 |
| `C2-qc-3x` | 24张 | 72张筛选后合成图 | 96张 | 12 | 480 |

每类核心真实图固定为：

```text
12名受试者 × pose02 1张 = 12张
```

`C1-raw-3x`、`C2-qc-3x` 每类合成图固定为：

```text
pose02：36张
```

因此每轮为：

```text
每类真实12张 + 合成36张；两类96张；batch size 8；12个优化步
```

`C0-pose02` 使用仅真实 pose02 采样协议。`C1-raw-3x`、`C2-qc-3x` 均保留相同的24张核心真实图，并对每个类别每轮各抽36张 pose02 合成图，40轮共1,440次曝光；每类1,500张池在本轮中不发生重复抽样。

## 9. 评价

### 9.1 评价层级

- 图片级：所有测试图片；
- 姿势级：仅报告 `pose02`；
- 受试者级主结果：每名受试者仅有1张 `pose02`，其预测概率即为受试者概率，阈值固定为0.5。

### 9.2 指标

主指标：

```text
subject-level Balanced Accuracy
```

关键次指标：

```text
subject-level MCC、Macro-F1、ROC-AUC、PR-AUC、Sensitivity、Specificity
```

同时输出混淆矩阵、逐图预测、逐姿势预测、逐受试者预测、采样历史与生成/QC审计清单。

## 10. 判定规则

| 问题 | 比较 | 初步支持条件 |
|---|---|---|
| 合成扩增训练方案是否有效 | `C1-raw-3x - C0-pose02` | BA、MCC均为正，且ROC-AUC/PR-AUC无明显反向下降 |
| 筛选是否有效 | `C2-qc-3x - C1-raw-3x` | BA、MCC均为正，且ROC-AUC/PR-AUC无明显反向下降 |

解释情形：

1. `C1-raw-3x > C0-pose02` 且 `C2-qc-3x > C1-raw-3x`：扩增训练方案和筛选均有正向信号；
2. `C1-raw-3x ≤ C0-pose02`，但 `C2-qc-3x > C1-raw-3x` 且 `C2-qc-3x > C0-pose02`：原始扩增训练方案无效，筛选后扩增训练方案有正向信号；
3. `C2-qc-3x ≤ C1-raw-3x`：当前QC未证明优于等量随机候选；
4. `C1-raw-3x ≤ C0-pose02` 且 `C2-qc-3x ≤ C0-pose02`：停止当前路线，不继续扩大候选规模。

单一 `fold_0 + seed22` 的结果仅用于决定是否进入正式验证，不形成最终论文结论。

## 11. 正式验证门槛

只有在 `C2-qc-3x` 对 `C0-pose02` 和 `C1-raw-3x` 均出现正向探索信号时，才进入：

```text
第一步：五折 × seed22
第二步：仅对 C0-pose02、C1-raw-3x、C2-qc-3x 补 seeds 23、24
```

正式结论基于五折受试者级OOF结果、同种子配对差值和受试者级配对bootstrap置信区间。不得在五折OOF结果后修改候选规模、筛选阈值、提示词、生成参数或分类参数。

## 13. 结论边界

本实验检验当前单中心、小样本、pose02 手部数据中的内部交叉验证效用。即使结果为正，也不能替代新的独立真实受试者外部测试；生成数据不会增加临床标签真值或证明临床诊断能力。
