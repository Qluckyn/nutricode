# C2-QC：基于手背营养表型的合成图像筛选实验规划书

## 1. 研究目的

在已完成的五折 `C0-real-step` 与 C1-noCN 消融基础上，验证：在合成图数量、类别比例、姿势、生成强度、来源受试者配额和分类训练预算完全一致时，基于结构质量与虎口—第一骨间肌相关表型的质量控制（C2-QC），是否优于分层随机抽取的合成图（C1-raw）。

本阶段只针对下列两条主生成路线：

```text
DataDream-I2I-noCN
FoundHand-DataDream-I2I-noCN
```

不重新比较 SD2.1 与 SDXL；它们保留为已完成的生成路线消融对照。

## 2. 固定实验条件

| 项目 | 固定设置 |
|---|---|
| 数据划分 | 五折受试者级划分，seed22 |
| 姿势 | 仅 pose02 |
| 测试集 | 仅使用各折真实 test 图；绝不参与生成、QC 或阈值设定 |
| 真实训练曝光 | 每 epoch 每类 12 张 |
| 合成训练曝光 | 每 epoch 每类 36 张 |
| 真实:合成 | 1:3 |
| 分类训练 | 40 epoch、batch size=8、约 480 updates、seed22 |
| 主要指标 | BA、MCC、F1、敏感度、特异度；Acc 为辅助 |

## 3. 候选池扩充

### 3.1 目标规模

每条路线、每折、每类建立 300 张候选：

```text
300 张/类 = 强度 0.15、0.22、0.30 各 100 张/类
```

两条路线、五折、两类总候选为 6000 张。现有每类 90 张候选保留并纳入候选池；因此需要新增 4200 张候选。

### 3.2 生成冻结规则

- 沿用已冻结的类别提示词、负向提示词、分辨率、推理步数与三档去噪强度；
- 沿用当前 noCN 设定：不使用 OpenPose-ControlNet；
- DataDream 路线使用类别 DataDream LoRA；
- FoundHand 路线使用 FoundHand 锚图与类别 DataDream LoRA；
- 外观父图和随机种子按训练受试者均衡轮换；FoundHand 路线的目标结构条件也按训练受试者均衡轮换；
- 仅使用当前 fold 的真实训练受试者；不读取 test 数据。

## 4. C2-QC 筛选方法

### 4.1 第一阶段：硬门控

候选图满足以下条件才进入评分：

1. 可检测到左右两只手的完整关键点；
2. 手腕、手掌与手指未被裁剪，且不存在明显额外、缺失或融合手指；
3. pose02 结构误差低于由训练图确定的阈值：FoundHand 路线与其实际输入的目标结构关键点比较；DataDream-noCN 路线与其真实 I2I 外观父图的 pose02 关键点或由真实训练图构建的 pose02 规范比较；
4. 与真实父图及已保留候选不存在近重复；
5. 左右虎口 ROI 均可依据关键点稳定定位。

### 4.2 第二阶段：第一指蹼—第一背侧骨间肌投影区表型一致性

本研究中的“虎口 ROI”严格定义为第一指蹼—第一背侧骨间肌投影区，而非仅截取拇指和食指间狭窄的 V 字皮肤开口。医学营养体格检查可观察拇指侧手背骨间肌的凹陷或丰满程度；本研究仅将该区域作为图像表型一致性线索，而非独立临床诊断。

对每只手，以拇指 CMC 点、食指 MCP 点和腕点分别记为 `p_thumb_CMC`、`p_index_MCP`、`p_wrist`。定义初始中点 `m=(p_thumb_CMC+p_index_MCP)/2`，ROI 中心为：

```math
c = m + 0.15 (p_wrist - m)
```

即从第一指蹼中点向腕部轻微平移，以覆盖更接近第一背侧骨间肌腹的区域。ROI 为沿“拇指 CMC -> 食指 MCP”方向的定向椭圆；其长、短轴分别为 `0.55` 与 `0.35` 倍手尺度，手尺度定义为腕点到中指 MCP 点的距离。左右手分别提取，任一侧 ROI 无法稳定定位则不通过硬门控。

在正式评分前，须在每折真实训练图中随机抽取 20 至 30 张叠加显示 ROI，并进行人工覆盖核验；核验通过后冻结上述几何参数，不再依据测试结果调整。

对当前 fold 的真实训练图，分别计算两类 ROI 特征原型：

```text
mu_mal：营养不良类真实训练图 ROI 原型
mu_normal：营养正常类真实训练图 ROI 原型
```

对生成图 x、其目标类别 y，定义：

```math
S_phenotype(x, y) = cos(z_ROI(x), mu_y) - cos(z_ROI(x), mu_not_y)
```

其中 `z_ROI(x)` 为左右 ROI 特征的平均向量。分数越高，表示该生成图更接近目标类的真实第一指蹼—第一背侧骨间肌投影区表型，且更远离相反类别。

### 4.3 第三阶段：综合质量评分

```math
Q(x, y) = 0.40 S_structure + 0.35 S_phenotype + 0.15 S_realism + 0.10 S_novelty
```

- `S_structure`：关键点完整性、pose02 一致性和双手结构质量；
- `S_phenotype`：上述 ROI 类别表型一致性；
- `S_realism`：与同类真实训练图整体特征分布的合理接近度；
- `S_novelty`：远离父图及已选候选的近重复程度。

所有评分器的原型、阈值和特征统计仅由当前 fold 的真实训练图及候选图计算。

### 4.4 分层公平选择

在“类别 × 生成强度 × 外观父受试者”分层内选择：

- C1-raw：按相同层配额随机抽取；
- C2-QC：按 Q 分数排序，并使用贪心多样性选择抽取；
- 两组最终均为 90 张/类，即每个强度档 30 张/类；
- 两组使用相同的父受试者配额与最大选取上限。

因此 C1 与 C2 唯一设计差异为随机抽取或 QC 筛选。

## 5. 分类比较

每条主路线分别构建：

```text
C1-DataDream-raw       vs C2-DataDream-QC
C1-FoundHand-raw       vs C2-FoundHand-QC
```

注意：现有 90 张候选池的 C1 结果不能直接作为上述对照。候选池扩充后，必须从同一个 300 张/类池重新构建并重跑 C1-raw。

主比较为同折、同路线的：

```text
Delta_BA  = BA(C2)  - BA(C1)
Delta_MCC = MCC(C2) - MCC(C1)
Delta_F1  = F1(C2)  - F1(C1)
```

同时报告敏感度与特异度变化，避免单一类别获益掩盖另一类别下降。

## 6. 执行顺序

1. 固化当前生成与分类配置；
2. fold_0 扩充候选并运行 QC 冒烟验证；
3. 审计候选数量、强度配额、父图配额和训练隔离；
4. 构建 fold_0 的配对 C1-raw/C2-QC 并验证训练入口；
5. 冻结 QC 阈值、权重和代码；
6. 对 fold_1 至 fold_4 重复相同流程；
7. 汇总五折均值、标准差、逐折配对差值；
8. 对表现最佳路线补 seed23、seed24，作为正式稳健性验证。

## 7. 合理性与准确性审查

### 合理性

- 300 张/类候选池约为最终选择数量的 3.3 倍，可提供筛选空间，同时避免不必要的大规模生成；
- C1/C2 同源候选、同数量、同配额和同训练步数，能够将差异主要归因于 QC；
- 虎口/第一骨间肌 ROI 与营养体格检查中关注的骨间肌丰满度方向一致，具有医学可解释性；
- 结构、表型、真实性与多样性联合评分避免只挑选“最像父图”或“最容易分类”的图像；
- 使用 C0-real-step 而非 C0-real-epoch 作为主 baseline，控制了更新步数混杂。

### 必须遵守的边界

- ROI 表型分数不是临床营养不良诊断，也不应被表述为临床金标准；
- QC 阈值和原型不得使用测试图；
- 先在 fold_0 验证代码，但冻结后不得根据其他折测试结果调权重或提示词；
- 每折测试样本较小，五折 seed22 只能构成初步证据，正式结论需增加分类随机种子；
- 若某分层的硬门控候选不足 30 张，必须记录并补生成同层候选，不得以其他层替代。

## 8. 预期论文表述

> 提出一种融合双手结构完整性、虎口—第一骨间肌相关表型一致性、真实性和多样性的合成手背图像质量控制方法。在相同合成图规模与训练预算下，验证其相对分层随机选择的下游分类收益。
