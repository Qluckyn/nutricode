# 手部合成数据扩展与分类验证任务规划书

## 1. 项目目标

在严格隔离真实测试受试者的前提下，复用 `sd_lora`、`generate` 和 `classify` 现有流程，为手部营养状态分类生成可控、可追溯的合成训练图像，并验证合成数据是否能提升真实受试者上的分类性能。

本项目不以“生成图片数量增加”为成功标准，而以受试者级交叉验证中，`真实数据 + 质量控制后的合成数据` 相比真实数据基线取得稳定提升为最终标准。

## 2. 任务范围

### 2.1 包含内容

- 按营养状态和姿势准备 LoRA 训练数据。
- 训练四套手部专用 LoRA。
- 使用对应 LoRA 和姿势 prompt 生成合成图。
- 对合成图执行解剖、姿势、分布及近重复质量控制。
- 将四种生成结果合并成分类所需的两个营养状态类别。
- 比较真实数据、常规增强、原始合成数据及 QC 合成数据的分类效果。
- 最终采用受试者级五折交叉验证，并报告受试者级指标。

### 2.2 不包含内容

- 不使用测试集训练或微调生成模型。
- 不使用测试集选择 prompt、LoRA epoch、生成参数或 QC 阈值。
- 不把同一受试者的两种姿势拆分到训练集和测试集。
- 不以生成图上的分类准确率替代真实测试集结果。

## 3. 数据与标签定义

### 3.1 生成阶段的四个复合类别

| 类别 | 营养状态 | 姿势定义 |
|---|---|---|
| `malnourished_hand_pose01` | 营养不良 | 双手握拳，手背朝向相机 |
| `malnourished_hand_pose02` | 营养不良 | 双手平放展开，手背朝向相机 |
| `normal_hand_pose01` | 营养正常 | 双手握拳，手背朝向相机 |
| `normal_hand_pose02` | 营养正常 | 双手平放展开，手背朝向相机 |

### 3.2 分类阶段的两个类别

生成完成后，四类按营养状态合并为：

```text
malnourished_hand
normal_hand
```

文件名保留姿势、生成配置和序号，例如：

```text
pose01_seed22_cfg001_000001.png
pose02_seed22_cfg001_000001.png
```

### 3.3 数据隔离原则

- LoRA、生成和分类训练只能使用当前折的真实训练受试者。
- 当前 `split_seed22` 可作为首轮流程验证或五折中的一折。
- 正式五折实验中，每一折必须独立训练 LoRA、独立生成图片、独立训练分类器。
- 禁止用包含其他折测试受试者的 LoRA 权重生成当前折训练数据。

### 阶段 D：合成图质量控制

#### 任务

1. 文件与尺寸检查：排除损坏、空白和异常尺寸图片。
2. 解剖检查：排除单手、三只手、缺指、多指、融合手指、严重变形和明显裁切。
3. 姿势检查：排除手掌朝上、pose 混淆及双手重叠异常。
4. 分布检查：使用固定的预训练特征提取器计算合成图与真实训练图的距离，排除离群图。
5. 近重复检查：排除与真实训练图或其他生成图过度相似的样本。
6. 类别捷径检查：比较两类图像的背景、曝光、分辨率、肤色和构图分布，防止类别与非临床因素绑定。
7. 人工盲审抽样：隐藏类别和生成配置，记录合格率和主要失败类型。
8. 输出 `raw` 与 `qc` 两套数据，保留所有拒绝原因，不直接删除原始生成图。

#### 验收标准

- 每张图片都有 `accepted/rejected` 状态和原因。
- QC 规则、阈值和模型版本固定且不依赖测试集。
- 两个营养状态、两个姿势的最终数量可统计并尽量均衡。
- 能生成 QC 前后质量报告和示例网格。

### 阶段 G：受试者级五折交叉验证

#### 任务

1. 将全部真实受试者按营养状态进行受试者级五折划分。
2. 每名受试者的 pose01、pose02 必须始终处于同一折。
3. 每名受试者恰好作为一次测试受试者。
4. 每折独立完成 LoRA 训练、生成、QC 和分类训练。
5. 汇总五折预测，不把同一受试者的两张图片当成两个独立受试者。

#### 受试者级融合

默认对同一受试者两个姿势的营养不良概率取平均，再形成最终受试者预测。姿势级结果作为辅助分析单独报告。

#### 主要指标

- sensitivity；
- specificity；
- balanced accuracy；
- macro F1；
- ROC-AUC；
- 五折均值、标准差和汇总混淆矩阵；
- 各姿势单独指标与双姿势融合指标。

#### 验收标准

- 所有折均通过受试者泄漏检查。
- 所有折的模型、数据、生成图和预测均可追溯。
- 报告包含每折结果、汇总结果和失败案例分析。

## 7. 风险与应对

### 7.1 小样本 LoRA 记忆

风险：每个营养不良姿势可能只有约 12 张图片，LoRA 可能复制训练样本。

应对：降低训练 epoch、首轮不训练文本编码器、执行最近邻检查、保留生成 seed，并比较 40/80/120 epoch。

### 7.2 手部解剖错误

风险：文本到图像模型容易产生多指、缺指和双手数量错误。

应对：使用强姿势 prompt、negative prompt、自动检测和人工抽检；若纯文本控制不足，再评估 img2img 或 ControlNet，不能仅继续堆叠 prompt。

### 7.3 类别捷径

风险：生成模型可能把背景、光照、年龄、肤色或皱纹当成营养状态。

应对：两类使用相同场景模板和生成参数；检查非临床属性分布；避免使用 `elderly`、`diseased`、`skeletal` 等极端词。

### 7.4 数据泄漏

风险：LoRA 或 QC 使用测试受试者会导致虚假提升。

应对：所有阶段以 fold manifest 为唯一数据来源；训练前和评估前均运行泄漏校验。

### 7.5 固定测试集过小

风险：单折营养不良测试受试者少，单个错误会显著改变结果。

应对：首折只验证流程，最终结论以受试者级五折汇总为准。

### 7.6 计算成本

风险：五折、四类意味着最多训练 20 套 LoRA，且包含多组合分类实验。

应对：按“单类冒烟测试 → 首折四类 → 首折分类验证 → 五折”逐级放大；只有上一阶段通过工程验收才进入下一阶段。

## 8. 里程碑与停止条件

### M5：五折验证完成

完成所有折并输出受试者级汇总结果。

### 项目成功条件

在五折受试者级评估中，`真实数据 + QC 合成数据` 相比 `仅真实数据` 和 `真实数据 + 常规增强` 表现出方向一致、具有实际意义的改善，且提升不是由单一折、单一姿势或明显类别捷径造成。

### 项目停止或调整条件

- 多轮调参后生成图解剖合格率仍过低；
- 生成图与真实训练图存在大量近重复；
- 仅合成训练在真实测试受试者上接近随机，且混合训练持续降低性能；
- 提升只存在于图片级而不存在于受试者级；
- 提升依赖背景、曝光或其他非营养状态特征。

触发这些条件时，应优先转向真实图轻量增强、预训练表征、受试者级正则化或受控 img2img，而不是继续增加纯文本生成数量。

## 9. 规划合理性检查

### 9.1 检查结论

本规划总体合理，可以作为项目实施主线，原因如下：

1. 数据隔离贯穿 LoRA、生成、QC 和分类全流程，避免最严重的测试泄漏。
2. 四个复合类别与现有 `sd_lora` 的逐类别 LoRA 训练结构匹配，首轮改造成本可控。
3. 将 pose 与营养状态显式拆开，有利于控制生成构图并诊断各姿势效果。
4. 先首折验证工程链路，再扩展五折，能控制计算成本。
5. 同时保留 raw 与 QC 合成数据，可以验证质量控制本身是否带来收益。
6. 最终采用受试者级而非单纯图片级指标，符合两张姿势来自同一受试者的数据结构。

### 9.2 必须坚持的修正点

规划执行时必须避免以下简化，否则结论不再可靠：

- 不能用全部真实数据训练一次 LoRA，然后复用到五折；每折必须独立训练。
- 不能因正常组有更多数据就默认必须永久下采样到 12 人；12 对 12 是首轮受控实验，后续应增加“正常组全部训练数据、总更新步数对齐”的消融。
- 不能把固定 40 epoch 视为最优值；它只是不会立刻采用原人脸 240 epoch 的保守起点。
- 不能用测试结果选择 QC 阈值、生成 prompt 或 LoRA checkpoint。
- 不能直接沿用 `Resize + RandomCrop`，横向双手图必须使用保持宽高比的补边预处理。
- 不能把五折均值简单解释为外部临床泛化能力；样本规模仍小，最终结论应表述为内部验证结果。

### 9.3 尚需在实施时确认的事项

- 实际受试者总数、每类数量及文件扩展名，以 manifest 扫描结果为准。
- 正常组首轮匹配抽样是否需要控制年龄、性别、肤色或采集批次；若有元数据，应采用分层匹配，而不是纯随机抽样。
- 当前环境中 `diffusers`、`peft`、`transformers` 与 SD2.1 权重的兼容性。
- 自动手部关键点工具对握拳 pose01 的可靠性；若不足，应保留人工盲审作为最终解剖 QC。
- 五折实验的算力预算；若无法承担 20 套 LoRA，应预先缩减消融组合，但不能破坏每折数据隔离。

综合判断：规划在方法学和工程路径上可执行，但最终可信度主要取决于“每折生成模型独立训练”“测试集不参与任何选择”以及“受试者级汇总”三项约束是否被严格落实。
